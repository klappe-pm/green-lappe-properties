## Lessons learned

- A folder move is only half a migration. The files moved on disk weeks ago, but
  because the IA contract (`information-architecture.md`), the orientation map
  (`README.md`), the instruction file (`CLAUDE.md`), and the enforcement script
  (`evaluate-ia-change.mjs`) were not updated in the same change, the repo is now
  in a state where the validator enforces a structure that no longer exists. This
  is the concrete failure mode the requested automation must prevent: **moves and
  their contract/index/enforcement updates must land atomically.**
- Grouping by function surfaced a real bug (broken Sanity imports) that the flat
  layout hid. Reorg is not cosmetic when configs use relative imports.
