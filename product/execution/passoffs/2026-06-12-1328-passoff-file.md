# Summary

Cleaned up the project documentation after the global Claude
`new-project-scaffolding` skill and `/new-project-scaffolding` command were
installed. The generic new-project scaffold prompt now lives under
`docs/prompts/`, the prior Lapsauskas-only prompt is removed, and the root
README points users at the global command/skill as the executable workflow.

renamed from `green-pm-style-guide.md` to `green-pm-design-system.md`. The
historical passoff deletion that was present in the worktree was restored so
passoff history remains intact.

# Key decisions

- Preserved `docs/passoffs/2026-06-07-1648-passoff-file.md` rather than
  staging its deletion. Passoffs are historical handoff records, not active
  prompt inventory.
- Kept historical passoff references to old paths unchanged. Those files record
  what was true when they were written.
- Treated `docs/prompts/` as the active home for the reusable prompt artifact
  and the global `.claude` skill/command as the execution surface.
- Removed untracked `Untitled.md` and `_index` junk artifacts from the working
  tree before staging; they were not committed.

# Risks and open questions

  handoff bundle file. References inside the handoff bundle that point to
  `green-pm-style-guide.md` are intentionally left alone unless they refer to

# Lessons learned

- The prompt-specific cleanup needed both path-level cleanup and index cleanup:
  moving the generic prompt into `docs/prompts/` without updating the root
  README and prompt index would have left two competing prompt homes.
- Historical passoffs should be preserved even when the active file they
  describe is superseded or removed.
