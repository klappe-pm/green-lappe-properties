# Key decisions

- Put the prompt in `docs/prompts/` rather than `docs/uxr/prompts/` because it
  is a repo/project scaffolding prompt, not a research reproducibility prompt.
- Kept the prompt prescriptive about process and IA but conservative about
  business content: it instructs the next agent to create placeholders and
  blocked decisions rather than inventing Lapsauskas-specific launch facts.
- Included Green repo source files to read in the prompt so the future scaffold
  session can ground itself in actual project history instead of relying on this
  passoff alone.
- Did not stage ignored local `.claude/` tooling. This checkout's ignored IA
  evaluator was adjusted locally so the required staged IA validation can accept
  the new `docs/prompts/` zone, but `.claude/` remains outside the tracked diff
  after the repo's prior untracking work.

# Risks and open questions

- Because `.claude/` is globally ignored and local-only in this checkout,
  another fresh checkout with an older local IA evaluator may need its local
  prompt-zone allowlist updated before validating future `docs/prompts/`
  changes.
- The prompt does not create the Lapsauskas repo; it is only the copyable
  scaffold instruction requested for later use.
- The Lapsauskas scaffold prompt intentionally leaves owner, entity, geography,
  financing, legal, insurance, trust-accounting, domain, brand, vendor, and
  launch decisions as unknown until the user provides facts.

# Lessons learned

- `docs/prompts/` needed a small README map update because the repo previously
  treated research prompts under `docs/uxr/prompts/` as the only prompt class.
- The IA evaluator's non-live reference guard also catches prompt examples that
  mention the archive path in backticks, so the scaffold prompt describes the
  archive area without linking to or depending on it.
