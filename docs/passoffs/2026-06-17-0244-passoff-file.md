# Key decisions

- **Built all routes in-repo under `src/green-pm-site/`**, feature-organized,
  per the user's "this is the project" direction.
- **Design tokens copied, not re-authored**; the design system stays the source
  of truth; Tailwind preflight disabled so `base.css` is the only reset.
- **External integrations stubbed behind explicit boundaries** rather than
  faked: forms render with disabled submit + `mailto` fallback (no dropped
  leads); portals show a "sign-in not connected" preview banner with
  Rentvine-shaped sample data; rentals/blog/faq use sample data shaped after the
  Sanity schemas.
- **Pre-public boundary enforced**: `noindex` default on every page +
  `robots.txt` disallow; portals disallowed; no real PII; no live submissions.
- **Legal pages shipped as drafts pending counsel** (no invented legal text).
- **Merged each increment after CI green** (build-test + evaluate-ia), per user
  instruction to merge and keep building.

# Risks and open questions

- The site is verified by `astro build`/`astro check`/`vitest` but has **not
  been run in a browser**; visual/responsive QA and a Lighthouse pass are still
  needed (performance budgets live in `80-system-architecture.md`).
- All dynamic content is **sample data**; nothing reads from Sanity/Rentvine yet.
- Forms do not submit anywhere; lead capture is currently `mailto` only.
- `greenpmpnw.com` is the canonical domain in code, but the domain purchase,
  Cloudflare Pages project, DNS, and Clerk/Rentvine/Sanity/HubSpot/Postmark
  accounts are all external/blocked.
- The broader original ask still has open doc deliverables (see Next actions):
  consolidated build/test/delivery plan, feature specs, test plan, agentic
  development design, and pointer-file updates — not yet written.

# Lessons learned

- Reset onto current `origin/main` between increments kept each PR a clean,
  fast-forwarding diff and avoided re-surfacing the earlier stale-base problem.
- Pre-verifying with the repo's own IA evaluator + `astro` toolchain before push
  made every CI run pass first try.
