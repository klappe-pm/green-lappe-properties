# Key decisions

- Kept the IA evaluator project-local under `.claude/` because it is a repository workflow guard, not product code.
- Made the pre-commit command evaluate staged content with `git show :path` so it checks what will actually be committed.
- Scoped strict filename cleanup to newly added, copied, or renamed docs so legacy filenames are not forced into unrelated rewrites.
- Required local README/index updates when adding launch, opportunities, or design-system docs because those folders already have canonical indexes.
- Did not stage or modify unrelated design-system files outside the `.claude` IA work.

# Risks and open questions

- The evaluator is deterministic and rule-based; it catches IA structure drift, not content quality.
- The sensitive-evidence check is path-based and conservative, but it is not a secrets scanner.
- Existing legacy filenames with spaces remain allowed unless they are added, copied, or renamed in a future change.
- Current working tree has unrelated unstaged and untracked files under `docs/design-system/` that should be reviewed separately before any future broad staging command.
