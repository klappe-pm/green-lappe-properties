# Key decisions

- The v3 target is not a new decision: `green-pm-tokens.css` v3.0.0 is the source of truth (Geist display/UI/figures, Newsreader prose, Fraunces signature-only, mono family removed). The style guide was made to match it.
- Moved the three `Megan` signature examples from `font-display` to `font-accent`. Rationale: after remapping the Tailwind `display` family to Geist, `font-display italic` would render Geist italic, not Fraunces; the signature must stay Fraunces.
- Reframed §12.3 optical sizing around Newsreader and Fraunces. Rationale: Geist has no `opsz` axis, so the old Fraunces-display opsz table no longer applies.
- Left `green-pm-tokens.css` untouched (verified already v3) and made no color, spacing, or component changes.

# Lessons learned

- When the implemented tokens are already migrated, reconciliation is mechanical-but-careful: make the prose match the shipped `tokens.css`, not re-decide the system. The one non-obvious correctness step was the `font-display` → `font-accent` signature fix — a blind "Fraunces"→"Geist" replace would have silently rendered the operator signature in the wrong face. No new `docs/lessons/` entries.
