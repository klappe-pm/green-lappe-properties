# Key decisions

- The plan lives under `docs/launch/plans/` because website domain, DNS, and internal launch infrastructure are launch-execution artifacts.
- `greenlappe.com` remains the preferred primary candidate, based on existing project docs and a Verisign WHOIS no-match snapshot at `2026-05-21T01:36Z`; it must be re-checked at purchase time.
- Internal testing should use protected staging and PR preview deployments before any public website launch.
- Cloudflare Registrar/DNS/Pages/Access is the default recommendation for a lightweight/static site, with a documented caveat that Cloudflare Registrar domains must remain on Cloudflare nameservers.

# Risks and open questions

- Domain availability can change before purchase.
- A domain can be available but still require trademark, trade-name, or market-confusion review.
- Hosting provider access-protection rules differ by plan and by preview versus production domain.
- The website implementation location is still open: current repo subfolder versus separate website repo.
- The public launch remains blocked by licensing/counsel-approved language and later content/style/asset decisions.
