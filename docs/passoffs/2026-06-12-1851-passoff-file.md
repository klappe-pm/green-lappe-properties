1. **`docs/` was relocated into a new `product/` harness.** On current `main`,

   `product/design/uxd/`, `docs/launch/` → `product/launch/`,
   `docs/gtm/` → `product/gtm/`, `docs/branding/` → `product/branding/`). Only
   `docs/passoffs/` and `docs/_archive/` remain under `docs/`.
2. **The IA is now enforced by a tracked evaluator + workflow** that did not
   exist on the stale base: `.claude/evaluate-ia-change.mjs`,
   `.claude/information-architecture.md`, `.claude/reconcile-tracking.sh`, and
   `.github/workflows/ia.yml`. (My earlier "missing `.claude/` IA tooling"
   blocker was an artifact of the stale base — the tooling exists on `main`.)
3. **Allowed top-level zones** (from the evaluator): `README.md`, `AGENTS.md`,
   `CLAUDE.md`, `WARP.md`, `GEMINI.md`, `LLM-BREADCRUMBS.md`, `.claude/`,
   `.githooks/`, `.gitignore`, `.github/`, `docs/`, `product/`, `prompts/`,
   `src/`, `scaffolding/`. **A top-level `site/` is not allowed** — that was the
   failure.
4. **`product/features/` already exists** as a canonical product zone (alongside
   `epics/`, `roadmaps/`, `opportunities/`, `program/`, `projects/`,
   `execution/`). Feature-centric planning has a home already.
5. **`AGENTS.md` is now the canonical project-local instruction file**;
   `CLAUDE.md`/`WARP.md`/`GEMINI.md` are meant to be thin pointers to it. This
   reframes the original "update all CLAUDE.md pointer files" request.
6. **`src/` is the IA's zone for executable project code** ("not documentation").
   The Astro site belongs there.

## The fix applied

Relocated the partial Astro scaffold from the disallowed top-level `site/` zone
into `src/green-pm-site/` (an allowed `src/` zone). Updated the scaffold README's
internal paths. No allowlist edit was needed — the relocation conforms to the
existing IA, which is the lower-risk fix than adding a new top-level zone.

# Lessons learned

- Always fetch and diff against current `origin/main` before a repo review —
  basing analysis on the checked-out branch's base missed a full docs→product
  restructure and the addition of enforced IA tooling.
- The IA evaluator only flags the PR's net additions when run on the merge ref,
  which is why only `site/` (not the stale-base reversion) appeared in CI.
