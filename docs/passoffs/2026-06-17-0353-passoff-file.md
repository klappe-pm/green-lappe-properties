# Key decisions

- **Browser tests run in CI, not the sandbox** (the Playwright/Lighthouse CDN is
  blocked locally). Local coverage is unit + static-HTML over `dist/`.
- **Real bugs fixed, not masked** — every E2E/Lighthouse failure was diagnosed to
  the offending element and fixed (see report).
- **Backend wired as an env-driven seam with graceful fallback** rather than
  faked — Sanity goes live on config; forms/portals keep honest "not connected"
  states until credentials exist (no invented secrets, per CLAUDE.md).
- **Cloudflare `_redirects`** added for production-native 301s.
- **Biome scoped to TS/JS/JSON**; Prettier keeps `.astro`/CSS.

# Risks and open questions

- Not yet verified in a real browser by a human; CI produces per-device
  screenshots (`visual-screenshots` artifact) for that review.
- All dynamic content is sample data until Sanity is configured + populated.
- Forms/portals/email/domain/deploy are blocked on external accounts; the design
  system still has ~20 TBDs to resolve before those features go fully live.
- Pre-public boundary (`noindex` + robots disallow) must stay until counsel
  approves public language and licensing/firm status is confirmed.

# Lessons learned

- A static-HTML suite over `dist/` plus Playwright in CI gives strong coverage
  even when browsers can't run locally; the static contrast guards catch a11y
  regressions before the slower Lighthouse job.
- Resetting onto `origin/main` between every increment kept each PR a clean
  fast-forward and CI green-on-first-try (after the bug-fix rounds).
