- Used dated lowercase-kebab filenames for repo validation, with plain-English
  H1 titles and task titles for Obsidian readability.

  for project, epic, feature, priority, owner, dependency, and stable task ID.

# Lessons learned

- A task repository can satisfy both Obsidian usability and repo IA if the
  stable loading path is a folder and the individual Markdown files keep the
  repo-required dated lowercase naming.
- For unknown vault automations, stable inline IDs and plain Markdown fields
  are safer than assuming one plugin-specific dependency implementation.
